# astrbot_plugin_aria2_plus（aria2增强下载助手）

AstrBot aria2 RPC 下载管理插件。支持通过指令或自然语言（AI 自动调用）添加/查询/取消下载任务，支持自定义下载目录、下载完成主动通知、用户白名单，全部配置均可在 AstrBot 管理面板中直接修改。

- 插件目录名：`astrbot_plugin_aria2_plus`
- 当前版本：`2.4.0`
- 作者：pkq（GitHub: [@kppq66](https://github.com/kppq66)）
- 仓库地址：https://github.com/kppq66/astrbot_plugin_aria2_plus

## 功能特性

- 🔗 连接测试、添加下载、查看任务列表、查询进度、取消任务
- 📂 自定义下载保存目录：可在插件配置中设置全局默认目录，也可在单次下载时临时指定
- 🌐 支持配置代理地址：aria2 部署在需要代理才能访问的网络环境时，可为 RPC 请求单独设置 HTTP/SOCKS5 代理
- 🔔 下载完成 / 失败后自动主动通知发起下载的会话（QQ 好友/群等），无需用户再手动查询
- 🤖 内置 4 个 LLM Tool（Function Calling），支持用户用自然语言让 AI 自动完成下载相关操作
- 🔒 用户白名单：可指定仅允许特定用户使用下载功能，其余用户的指令/AI 调用都会被拒绝
- ⚙️ 所有配置项都通过 AstrBot 管理面板的"插件配置"可视化编辑，无需手动改文件、无需重载插件即可生效

## 安装

### 方式一：URL 安装（推荐）

在 AstrBot 管理面板 -> 插件 -> 右下角 `+` -> 选择"URL 安装"，填入：

```
https://github.com/kppq66/astrbot_plugin_aria2_plus
```

安装完成后，打开管理面板 -> 插件 -> aria2增强下载助手 -> 插件配置，按下方说明填写配置。

### 方式二：手动安装

1. 将本插件文件夹放入 AstrBot 的 `data/plugins/` 目录下，目录名保持为 `astrbot_plugin_aria2_plus`
2. 安装依赖（AstrBot 一般会自动根据 `requirements.txt` 安装，如未自动安装可手动执行）：

   ```bash
   pip install -r requirements.txt
   ```

3. 在 AstrBot 管理面板中重载插件
4. 打开管理面板 -> 插件 -> aria2增强下载助手 -> 插件配置，按下方说明填写配置

## 配置说明（插件配置面板）

从 2.3.0 版本起，配置不再通过手动编辑 `config.yaml` 完成，而是统一在 AstrBot 管理面板的插件配置界面中设置，保存后即时生效，无需重载插件。

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `aria2_url` | 字符串 | `http://127.0.0.1:6800/jsonrpc` | aria2 的 JSON-RPC 接口地址 |
| `token` | 字符串 | `""` | aria2 的 `rpc-secret`，未设置密钥则留空 |
| `download_dir` | 字符串 | `""` | 默认下载保存目录（aria2 所在机器/容器上的路径，不是 AstrBot 所在机器的路径）。留空则使用 aria2 自身默认目录 |
| `proxy` | 字符串 | `""` | 访问 aria2 RPC 接口使用的代理地址，例如 `http://127.0.0.1:7890` 或 `socks5://127.0.0.1:1080`。留空表示不使用代理，直连 `aria2_url` |
| `qq_notify` | 布尔 | `true` | 下载任务完成/出错后，是否主动推送通知到发起下载的会话 |
| `notify_interval` | 整数 | `10` | 后台轮询检测任务状态的间隔（秒）。值越小通知越及时，但请求 aria2 越频繁 |
| `allowed_users` | 列表 | `[]` | 允许使用下载功能的用户 ID 白名单，留空表示不限制，所有人都可以使用 |

> ⚠️ 主动通知依赖 AstrBot 的 `context.send_message` 能力，实际能否推送取决于你接入的消息平台是否支持主动消息发送（并非所有平台都支持）。

### 关于代理（proxy）

- 仅作用于插件访问 aria2 RPC 接口的请求，不影响 AstrBot 其他功能
- 支持 `http://`、`https://` 代理，直接可用
- 如需使用 `socks5://` 代理，需要额外安装 `PySocks`（可在 `requirements.txt` 中取消对应注释，或手动 `pip install requests[socks]`），否则会报错提示缺少依赖
- 留空则不使用代理，直接连接 `aria2_url`

### 关于用户白名单（allowed_users）

- 填写的是发送者的用户 ID（例如 QQ 号），可通过 AstrBot 内置的 `/sid` 指令获取
- 留空（默认）表示不限制，任何人都可以使用下载相关指令和 AI 工具
- 一旦填写了至少一个 ID，只有名单内的用户可以使用 `/下载`、`/下载列表`、`/下载状态`、`/取消下载` 以及对应的 AI 自动调用工具；不在名单内的用户会收到"⛔ 你没有权限使用下载功能"的提示
- `/aria2测试`（连接测试）不受白名单限制

## 指令列表

| 指令 | 说明 |
|---|---|
| `/aria2测试` | 测试与 aria2 的连接是否正常 |
| `/下载 <链接>` | 添加一个新的下载任务，使用插件配置中的默认目录（若已设置） |
| `/下载 <链接> <目录>` | 添加下载任务，并为本次任务指定自定义保存目录，优先级高于默认目录 |
| `/下载列表` | 查看当前正在进行的下载任务 |
| `/下载状态 <GID>` | 查询指定任务的详细进度 |
| `/取消下载 <GID>` | 取消（删除）指定的下载任务 |

## AI 自动调用（LLM Tool）

在正常对话中，用户无需输入上述指令，只需用自然语言表达意图（例如"帮我下载这个链接，存到 /downloads/movies"、"我的下载任务怎么样了"、"取消 GID 为 xxx 的任务"），模型会自动判断并调用以下工具：

| Tool 名称 | 功能 | 参数 |
|---|---|---|
| `aria2_add_download` | 添加下载任务 | `url`（下载链接），`dir`（可选，自定义保存目录） |
| `aria2_list_downloads` | 查看当前所有下载任务 | 无 |
| `aria2_get_status` | 查询指定任务的详细状态 | `gid` |
| `aria2_cancel_download` | 取消指定下载任务 | `gid` |

通过 AI 添加的下载任务同样会被纳入自动通知追踪，完成后会主动推送结果；同样受用户白名单限制。

## 已知限制

- 任务追踪（GID → 会话）目前保存在内存中，AstrBot 重启或重载插件会丢失映射；重载前已存在但未完成的任务不会补发通知
- `download_dir` / 单次下载指定的目录，指的是 aria2 服务进程所在机器上的路径；如果 aria2 部署在别的服务器或容器上，请填写该服务器上真实存在且 aria2 有写入权限的目录
- 需要 aria2 已开启 RPC（`--enable-rpc`），且网络可达

## 常见问题

**Q: 重载插件报错 `[Errno 2] No such file or directory: 'config.yaml'`？**
A: 已在 2.1.0 版本修复；2.3.0 版本进一步移除了对 `config.yaml` 的依赖，改为使用 AstrBot 插件配置面板，请更新到最新版本并在管理面板中重新填写配置。

**Q: 收不到主动通知？**
A: 请确认 `qq_notify` 为 `true`，且你所使用的消息平台/适配器支持主动消息推送。

**Q: 设置了 `allowed_users` 后自己也不能用了？**
A: 请确认填写的是正确的用户 ID（用 `/sid` 获取），并且 ID 之间没有多余的空格。
